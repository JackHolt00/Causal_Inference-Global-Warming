Let’s now create a regression line for the data either side of the cut off point to see if there is a noticeable difference and to inform which method of RDA should be applied here.

#first grouping the data fore pre-1840 and post-1840
yearly_temps_grouped  <- yearly_temps %>%  mutate(group = case_when( year < 1840 ~ "pre-1840", year >= 1840 ~ "post-1840"))

#creating the plots
yearly_temps_plots <- yearly_temps_grouped %>% ggplot(aes(x=year, y =avg_temp, color = group)) +
  geom_point() + theme_bw() + xlab("Year") + ylab("Change in Average Temperature over Time") + 
  geom_vline(xintercept = 1840, linetype = "dashed") +
  geom_smooth(aes(group = group), color = "black", method = lm)

yearly_temps_plots

The scatter plot with regression lines shows that there is not a large jump in the intercept with the cutoff point. However, the slope of the regression line switched direction at the cutoff point: before 1840, the regression line has a negative slope, whereas after 1840 the regression line becomes positive. This change in slope, rather than a change in intercept, calls for a “kink” RDA (learn more about kink RDA here. We can use rdrobust to test if the slopes either side of the cutoff (within a bandwidth) vary enough to suggest a causal effect.

Regression Discontinuity Analysis

The first step is to select the bandwidth we will be using around the cutoff point. The rdrobust package is sufficient to create an appropriate bandwidth.

bandwidths <- (rdbwselect(y=yearly_temps$avg_temp, x= yearly_temps$year, c = 1840)) #deriv = 1 indicates that this is for a kink RDA

bw <- bandwidths$bws[1][1] # store the bandwidth value as a variable

yearly_temps_plots + geom_vline(xintercept = 1840 + c(bw, -bw))

Performing RDA

The data is prepared and we can now carry out the analysis.

The difference in predicted values at the cutoff of 1840 from the local regressions on either side, using the chosen bandwidth was about 0.61 (for the bias corrected calculation)

The standard error of the this gives an indication of how meaningful this difference is, and gives an error of 0.15. Even when accounting for this error, the ATE will still not be near to 0 - suggesting that the treatment variable does lead to a jump in the outcome.

Finding the p-value will give a more decisive judgement of the results:

rda$pv
                      P>|z|
Conventional   1.270360e-03
Bias-Corrected 5.988216e-05
Robust         2.180083e-04

Assuming a threshold 0f 0.05, the p-value is lower than required and we can therefore conclude the the jump at the cutoff point created by the differing regression lines is statistically significant.

Conclusions

The results of the analysis show that there is a causal relationship between the Industrial Revolution and the rise in global temperatures. This result matches the models and predictions made by climate scientists that the rise of industry and the resulting excess greenhouse gases is causing global warming.

The results are a major cause for concern and should be used to inform corporations of the dangers of continual release of harmful pollutants.

That being said, it is impossible for such a study to analyse this data in isolation and assume that there are no other variables that could be affecting the global temperatures. The climate and global systems are extremely complex, and there are limits to the statistical inferences we can make from this data. For example, this dataset only extends as far back as the 1750s. Trends in the climate over vast geological time scales and ice age cycles should be taken into account to identify if the current trend is anomalous compared what we should expect in the current geological period.

In addition to the assumption that there are no other variables driving the global temperatures, we have no viable counterfactual model that we can use - we can not imagine an Earth where the Industrial Revolution did not happen.

These factors provide limits to our investigation, but the analysis does strongly suggest that the rise of industry led to increasing average global temperatures.
